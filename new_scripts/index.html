New LLDB Scripts

Useful scripts when you need objection

Published Fri, Jun 25, 2021 by XdaemonX

Introduction

On the last two pentests I had to work with application with short pentest duration. I did not have time to bypass RASP and things like that.

As I needed to traverse directories and inspect files, I have created four lldb scripts to do just that.

The scripts are:

  • appEnv
  • cat
  • ls
  • download

appEnv

Print the path the applications bundle directory as well as library directory.

cat

Print the output of file, mostly used for text files.

ls

List the content of directory.

You can pass special variables, instead of typing full path, they are:

  • lsAppDir/PATH/TO/THE/DIR - for directory in apps bundle directory
  • lsLibDir/PATH/TO/THE/Dir - for directory in apps library directory

download

Download file the application.

You need to pass host and port to the server where file will be uploaded as well as path to the file you want to download.

Right now, application is sending base64 representation of file so when you write your server make sure to have that included.

Base64 encoded file will be stored in form data inside ff parameter in request body.
